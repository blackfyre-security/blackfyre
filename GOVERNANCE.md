# Governance

Blackfyre currently uses a BDFL model: the project founder is the maintainer
and has final say on roadmap and merges. This is the honest state of a young
project, not the end state.

- **Deciding:** the maintainer merges PRs and sets the roadmap (see ROADMAP.md).
- **Becoming a committer:** sustained, quality contributions → invited as a
  committer with merge rights over the areas you work on.
- **Contributions:** everyone submits PRs under the DCO (see CONTRIBUTING.md).
  All merged code is Apache-2.0.
- **Conduct:** CODE_OF_CONDUCT.md applies to all project spaces.
- **Security reports:** see SECURITY.md — do not open public issues for
  vulnerabilities.
